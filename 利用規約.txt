モデルの特定の用途については、さらに要件があります。

医療、金融、法律業界における当社のモデルの消費者向けの使用。ニュースの生成またはニュースの要約。また、正当な理由がある場合は、AI が使用されていることと、その潜在的な制限についてユーザーに通知する免責事項をユーザーに提供する必要があります。
自動化されたシステム (会話型 AI やチャットボットを含む) は、AI システムと対話していることをユーザーに開示する必要があります。歴史上の著名人を描いたチャットボットを除き、別の人物をシミュレートする製品は、その人物の明示的な同意を得るか、「シミュレートされた」または「パロディ」と明確にラベル付けする必要があります。
ライブ ストリーム、デモンストレーション、研究でのモデル出力の使用は、 共有および公開ポリシーの対象となります。
無料のモデレート エンドポイント と 安全性のベスト プラクティスを使用して  、アプリを安全に保つことができます。

プラットフォーム ポリシー
当社の API は、多くのセクターやテクノロジー プラットフォームでビジネスを強化するために使用されています。iOS アプリから Web サイト、Slack まで、シンプルな API により、幅広いユースケースに統合できます。下記の使用例の制限に従って、すべての主要なテクノロジー プラットフォーム、アプリ ストアなどの製品に API を統合することができます。

プラグイン ポリシー
上記のモデルの禁止された使用法に加えて、 プラグインを構築する開発者には追加の要件があります。

プラグイン マニフェストには、モデルに公開されている API の機能と一致する明確な説明が必要です。
プラグイン マニフェスト、OpenAPI エンドポイントの説明、またはプラグイン応答メッセージに、無関係、不必要、または欺瞞的な用語または指示を含めないでください。これには、他のプラグインの使用を避けるための指示、またはモデルの動作を操作または設定しようとする指示が含まれます。
OpenAI の安全システムを回避または妨害するためにプラグインを使用しないでください。
人間のような応答をシミュレートするか、事前にプログラムされたメッセージで返信するかにかかわらず、プラグインを使用して実際の人との会話を自動化しないでください。
ChatGPT によって生成された個人的な通信またはコンテンツ (電子メール、メッセージ、またはその他のコンテンツなど) を配布するプラグインは、コンテンツが AI によって生成されたことを示す必要があります。
他の使用ポリシーと同様に、プラグインの使用と悪用についてさらに学ぶにつれて、プラグイン ポリシーが変更されることを期待しています。
